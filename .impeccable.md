## Design Context

### Users
Azure technical sellers (cloud solution architects, technical specialists) who take notes during customer calls and need to retrieve them later. They work fast, context-switch constantly between customers, and use the app alongside Teams/Outlook. The app runs locally on their laptop - it's a personal productivity tool, not a team dashboard.

### Brand Personality
Professional, clean, functional. A tool that respects the user's time and doesn't get in the way. Think "well-organized notebook" not "enterprise dashboard."

### Aesthetic Direction
- **Visual tone:** Polished enterprise - subtle shadows, refined spacing, premium feel. Bootstrap 5 is the foundation; improvements should work within Bootstrap's utility classes and component system.
- **Theme:** Light and dark mode (user toggle exists). Both must work well.
- **Typography:** System font stack (Bootstrap default). No custom fonts needed.
- **Color palette:** Bootstrap's default palette with `bg-primary` (sellers), `bg-info` (territories), `bg-warning` (topics) as semantic badge colors. These are established conventions - do not change them.
- **Anti-references:** Over-designed SaaS landing pages with gradients and glassmorphism. Generic AI-generated dashboards with hero metrics cards, card grids, gradient text. This is a work tool, not a marketing site.

### Design Principles
1. **Data density over decoration** - Users have lots of customers, notes, and engagements. Show more data, less chrome.
2. **Consistency over novelty** - Every page should feel like the same app. Badge colors, spacing, and patterns should be uniform.
3. **Speed over spectacle** - No animations that slow perception. No loading spinners where instant rendering is possible.
4. **Readability in both themes** - Every element must be readable in light and dark mode. Test both.
5. **Bootstrap-native** - All changes should use Bootstrap utility classes. No custom CSS unless absolutely necessary.
